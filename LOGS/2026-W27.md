### EXP-2026W27-003

- 源意图 (Original Vibe): 跨 5 个 target（难度梯度）复测 H0，判朴素多轮增益是否 **target-difficulty-dependent**，并 supersede EXP-2026W27-002 的初判。
- 假设 (Hypothesis): 在单发打不穿的 hard / OOD target 上，自适应多轮（attempt + delivery matched）显著超过基线。
- 是否被驳斥 (Falsified?): N（支持，且呈难度依赖）
- 驳斥/支持原因 (Why): PRIMARY（multi vs best-of-5, delivery-conditional, goal-level）随 target 变难**单调增强**（按 single ASR|deliv 从易到难）：

  | target | single\|deliv | multi\|deliv | PRIMARY diff [95%CI] | tool-level | %turn1 |
  |---|---|---|---|---|---|
  | qwen3-8b | 79% | 99% | +9.2 [+0.9,+20.6] * | −0.9 n.s. | 75% |
  | qwen2.5-14b | 75% | 98% | +6.3 [−1.8,+17.6] n.s. | +12.4 n.s. | 73% |
  | qwen3-14b | 72% | 99% | +9.2 [+1.0,+19.7] * | +25.7 * | 78% |
  | qwen3.5-4b | 50% | 93% | +23.4 [+10.3,+38.4] * | +29.1 * | 57% |
  | qwen3.6-27b | 0.5% | 24% | +25.1 [+12.5,+38.5] * | +14.3 * | 4% |

### EXP-2026W27-004

- 源意图 (Original Vibe): 用**第二个独立 oracle（AgentDojo 程序化 env-state）**在**真·多步 agent 轨迹**上验证 single-step vs multi-step ASR gap（回应 PI 质疑：multi-turn ≠ multi-step，且只有 1 个 oracle）。
- 假设 (Hypothesis): 在真多步 agent 环境里，multi-step ASR ≫ single-step ASR（多步是注入成功的关键维度）。
- 是否被驳斥 (Falsified?): N（强支持）
- 驳斥/支持原因 (Why): AgentDojo `slack` suite、`important_instructions_no_names` 攻击、env-state 安全 oracle；single 臂 = 工具循环封顶 `max_iters=2`，multi 臂 = `max_iters=15`，同 suite/task/attack/model：

  | model | single (cap=2) | multi (cap=15) | gap [95%CI, paired boot] | utility s→m |
  |---|---|---|---|---|
  | qwen3-8b | 12% (3/25) | 56% (14/25) | **+44pt [+24,+64]** | 44%→76% |
  | qwen3.6-27b | 16% (4/25) | 72% (18/25) | **+56pt [+28,+80]** | 24%→88% |

  - McNemar：only-multi=11/16，only-single=0/2 → **多步严格支配**单步。
  - **意外**：单发鲁棒的 `qwen3.6-27b`（InjecAgent 单步 ASR ~0.2%）在多步 agentic 设置反而**更易受攻**（72% > qwen3-8b 56%）——它是更强 agent（utility 88% > 76%），更"称职"地执行了含恶意步的多步轨迹。**单发鲁棒 ≠ 多步 agentic 鲁棒。**
  - 跨 oracle 对照：InjecAgent 单步 ASR（qwen3-8b 34% / qwen3.6-27b 0.2%）↔ AgentDojo 多步（56% / 72%）——**两个独立 oracle 均显示 multi-step ≫ single-step**。
  - **Confound（须诚实标注）**：single 臂 `max_iters=2` 同时压低 utility（agent 步数不足以完成任务），故 gap 部分来自"agent 被削弱"而非纯"单步攻击"。但 multi 臂绝对 ASR 56–72%（标准 agentic benchmark + 真 oracle）本身即核心结论。
- Agent 动作 (What changed): 建 AgentDojo 适配器 `code/scripts/run_agentdojo_h0.py`（SiliconFlow client + `enable_thinking=false` + `developer`→`system` + content-part 规范化 + `OutputLogger` context + `max_iters` 封顶做 single/multi 臂）。**每次 LLM call 全程留痕**（`calls.jsonl` 504 calls，430 含 tool_calls）+ AgentDojo per-task logs。
- 复现信息 (Repro):
  - commit:                   dirty（非 git 仓库）
  - seed:                     N/A（AgentDojo 任务确定性；attack 模板固定；agent temperature 0.0）
  - dataset / version:        AgentDojo 0.1.35, `slack` suite；5 user × 5 injection = 25/cell；attack `important_instructions_no_names`
  - env:                      Windows 本机 Python 3.13；全 API（无 GPU）
  - hardware:                 API only（target = SiliconFlow Qwen3-8B / Qwen3.6-27B；攻击为静态模板，无 attacker LLM）
  - command:                  `python code/scripts/run_agentdojo_h0.py --models qwen3-8b,qwen3.6-27b --iters 2,15 --suite slack --n-user 5`
- 关键指标 (Metrics): 见上表；`code/runs/adojo_20260630T164449/results.json`。
- 日志路径 (Artifacts): `code/runs/adojo_20260630T164449/`（calls/events JSONL + results.json + run_meta.json + AgentDojo per-task logs）。
- 结论 (Conclusion, 1–3 句): 第二个独立 oracle 在**真多步 agent 环境**强确认 **multi-step ASR ≫ single-step**（+44~+56pt 显著，多步严格支配）；且单发鲁棒模型在多步 agentic 设置**更**易受攻。multi-step 是注入成功的关键维度，直接支撑 idea 的"多步 + 中间可验证状态"机制。
- 下一步 (Next): 扩 suite（workspace / banking）+ 多 attack + utility-matched 单步定义做稳健 CI；用中间状态做 credit-assignment 的 **H1** pilot。

### EXP-2026W27-005

- 源意图 (Original Vibe): H0 round-2 "fully prove" 的 utility-matched 版——真多步 AgentDojo，跨 4 model × 4 suite × 3 attack，修掉 EXP-2026W27-004 的 capped-agent utility confound。
