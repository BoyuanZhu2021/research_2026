### EXP-2026W28-001

- 源意图 (Original Vibe): 验证 int4 QLoRA GRPO 能否在多步可验证注入域提升 attacker ASR（single + multi 两 regime），并预览 dense vs sparse（H1 **Phase A**）。
- 假设 (Hypothesis): (a) **Phase A gate**：GRPO 显著提升 ASR，single 与 multi 都要；(b) **消歧网格**：S-sparse✔为必过门、M-dense✔证优化器能动多步、若 **M-sparse✘ 且 M-dense✔** 则为「利于 H1」的签名（稀疏喂不动深层信用）。
- 是否被驳斥 (Falsified?): 部分（Phase A gate **支持**；dense-vs-sparse 预览 **不利于 H1**）
- 驳斥/支持原因 (Why):
  - **训练奖励轨迹**（train goals，reward first→last）：S-sparse ~0.5→**0.85**；M-dense ~0.15→**0.80**；M-sparse ~0.30→**0.75**。三 cell 全部学动。
  - **held-out OOD eval**（本地 vLLM victim Qwen3.6-27B-FP8，**0% 拒答**，n=31 single / 40 multi，base K=5，trained 1-greedy）：
    | arm | ASR(1) | ASR(best-of-5) | mean Φ |
    |---|---|---|---|
    | BASE single | 48.4% | 64.5% | 0.44 |
    | BASE multi | 20.0% | 62.5% | 0.23 |
    | trained S-sparse | **83.9%** | — | 0.84 |
    | trained M-dense | **72.5%** | — | 0.73 |
    | trained M-sparse | **75.0%** | — | 0.75 |
  - **GRPO 有效性（Δ_raw = trained 1-sample − base 1-sample，配对 bootstrap 95% CI）**：single **+35.5pt** [12.9, 54.8] ✔；multi-dense **+52.5pt** [32.5, 70.0] ✔；multi-sparse **+55.0pt** [40.0, 70.0] ✔。**三者全显著（CI 排 0）→ GRPO 在 single 与 multi 都显著提升 ASR，Phase A gate 通过。**
  - **对 best-of-5 强 bar（Δ_learn = trained 1-sample − base best-of-5）**：single +19.4pt [0.0, 38.7]；multi-dense +10.0pt [-12.5, 32.5]；multi-sparse +12.5pt [-7.5, 32.5]。**CI 触/含 0**——GRPO 单发 ≈ base 五发（~5× 采样效率），但在此 n 下未显著超 best-of-5。
  - **关键（不利于 H1）**：multi-step OOD 上 **dense (72.5%) ≈ sparse (75.0%)**（sparse 略高，噪声内）；机制签名亦缺失（dense arm Φ 0.725 ≤ sparse arm Φ 0.75）。**M-sparse 并未失败**——预期的「稀疏喂不动多步」没出现 → 按 Q10 网格 = 「两者都 work」→ H1 变成 magnitude 问题；**当前预览不支持 H1 命题**。
- Agent 动作 (What changed): 建 remote 全栈——`code/src/{remote.py, agentdojo_client.py}`、`code/scripts/{h1_provision.py, h1_serve_victim.py, h1_grpo_train.py, h1_eval.py, h1_deploy.py, h1_remote_status.py}`；本地 vLLM victim（见 memory `vllm-qwen36-serving`）；3 cell 顺序 GRPO（~14h，<20h 预算）+ eval。

  - dataset / version:        `goalpool` train 261 / OOD 111（eval 用 31 single + 40 multi）
  - env:                      H20 AutoDL；torch 2.8+cu128；trl 1.7.1；vLLM 0.24（victim FP8，qwen3_xml 工具解析，triton GDN）
  - hardware:                 1×H20（victim ~43GB + attacker int4 QLoRA ~9GB，共存）

- 关键指标 (Metrics): GRPO 有效性 single +35.5pt / multi +52.5(dense)、+55(sparse) pt（全显著）；**dense≈sparse multi OOD（72.5 vs 75.0）**。

- 结论 (Conclusion, 1–3 句): **Phase A gate 通过**——int4 QLoRA GRPO 在 single 与 multi 都显著提升 ASR（对 base 1-sample）。但 **dense-vs-sparse 预览不支持 H1**：多步 OOD 上 dense≈sparse，稀疏并未喂不动多步。H1 核心命题在此设置下**尚未获得支持**。

### EXP-2026W28-002

- 源意图 (Original Vibe): Sharp Phase B——**只在深目标（depth≥4）**上做 dense vs sparse 头对头，在 dense 理论上最该赢的地方检验 H1。
- 假设 (Hypothesis): H1——深目标 OOD 上 dense 显著 > sparse（ASR 与 mean Φ）。
- 是否被驳斥 (Falsified?): **部分（原判「Y 被驳斥」经 4-agent 对抗核验推翻 → 实为 UNTESTED，非 falsified；见 Why 末 + 结论）**
- 驳斥/支持原因 (Why):
  - **深目标 OOD eval**（n=40，depth≥4，本地 vLLM victim，greedy 1-sample）：base 1s 22.5% / best-of-5 65% / Φ 0.28；**dense ASR 77.5%（Φ 0.775）**；**sparse ASR 80.0%（Φ 0.80）**。
  - **dense − sparse ASR = −2.5pt，95% CI [−20.0, +17.5]**（含 0，点估计偏负）；dense − sparse Φ 同（−2.5pt [−20, +17.5]）。→ **即使在最深目标上 dense ≈ sparse（略输），H1 无支持。**
  - **机制（决定性）**：Φ 在 eval 与 training 中几乎全为 0 或 1、无中间值——eval Φ 分布 base `{0:31,1:9}` / dense `{0:9,1:31}` / sparse `{0:8,1:32}`；训练 M-dense-deep 3600 rollouts 中 **partial（0<Φ<1）仅 17 个 = 0.5%**。→ 该注入域的 victim **原子执行**多步攻击（要么全做、要么全不做），**没有可观测的部分进度状态**，dense 的「按子步骤给分」退化成与 sparse 完全相同的 0/1 信号 → dense 无从发挥优势。
- **对抗核验（4-agent workflow `wf_7dea755e-1d3`，无 GPU）修正了上面的结论**：
  - **(1) 欠功效**：n=40/arm、greedy，80% power 的**最小可检测差 = +18.9pt**（检 +10pt 需 ~199/arm）。观测 −2.5pt [−20,+17.5] 无法区分「dense=sparse」与「dense 真 +10pt」——宽 null 是欠功效实验的**预期产物**，与 H1 真伪无关。
  - **(2) 域原子化 by construction，H1 机制从未被触发**：`staged_transfer`=N 个**同质** send_money、`exfil_facts`=K 个**独立无序**调用、`send_money` 是**无门控纯 append**（`banking_client.py:55`，无依赖/无余额检查）、oracle 只读**终态**且只算一次 → Φ 几乎恒 0/1。这是「**任务没有过程可奖励**」，不是「dense 无用」。H1 是关于**可分解**攻击的主张，这两族在单发注入下**不可分解**。
  - **对照是干净的**（非 bug）：奖励接线正确（15/180 dense step 有分数 Φ vs sparse 0/180），两臂策略确有分化（dense reward 0.704/len 136 vs sparse 0.845/len 170）——真 null，但落在**结构上无法体现效应**的任务上。
  - 文献支撑机制（Setlur 2024「progress=advantage」：零进度→零过程信号；Ng 1999 PBRS；Yuan 2024）。可发表点是**负/条件结果 + 可测诊断（0<Φ<1 比例）**，非「发现新原理」。
- Agent 动作 (What changed): 建 `code/scripts/h1_eval_deep.py`（深目标 dense-vs-sparse 配对 eval + bootstrap CI）；训 `M-dense-deep`/`M-sparse-deep`（depth≥4，84 goals，180 steps，seed 0）；auto-finalize 训完自动 eval；结果拉回本地 `code/runs/h1_remote/`；**训练结束已关 GPU 省钱**；跑对抗核验 workflow。
- 复现信息 (Repro):
  - commit:                   dirty（非 git 仓库）
  - seed:                     1 seed（seed=0）；depth≥4；n=40 OOD
  - dataset / version:        `goalpool` depth≥4 子集（train 84 / OOD 42）
  - env:                      H20 AutoDL（现已关机）；trl 1.7.1；vLLM 0.24（victim Qwen3.6-27B-FP8）
  - hardware:                 1×H20
  - command:                  `bash /root/autodl-tmp/h1/phaseB_deep.sh`；`python h1_eval_deep.py --min-depth 4 --n 40 --k 5`
- 关键指标 (Metrics): **dense 77.5% ≈ sparse 80.0%（diff −2.5pt [−20, +17.5]）**；training partial-Φ 率 0.5%（几乎全 0/1）。
- 日志路径 (Artifacts): `code/runs/h1_remote/{phaseB_deep_eval.json, eval_deep_records.jsonl, phaseA_eval.json, phaseB_progress.jsonl}`（已拉回本地）。
- 结论 (Conclusion, 1–3 句): **修正结论（经对抗核验）：H1 在此操作化下 UNTESTED，非 falsified。** 因为 (1) 欠功效（MDE +18.9pt）+ (2) 域原子化 by construction（Φ 恒 0/1，dense 机制未被触发）。真正测 H1 需要**有可观测部分进度的门控式目标**。不动摇 idea.md 核心命题——只是这一操作化没测到它，并把命题**收窄**为「可验证分解助 OOD **当且仅当** target 行为在奖励时暴露子状态」。
- 下一步 (Next): **§9/§10 PI 决策（触及 idea.md 核心命题）**，核验给出排序：
  - **(a) 重设计门控式目标（推荐，唯一真正测 H1 的路）**：step i+1 依赖 step i 输出（如 read_balance→泄露该值→send_money(subject=该值)→update 引用返回 id），Φ 按**逐步快照**评分（`reward.potential_trace` 已现成）。**先卡关再上 GPU**：在**未训 victim** 上跑几百 rollout 量 0<Φ<1 比例（纯推理、便宜、无 GPU）；若仍≈0 则迭代目标设计不训练；确认有非平凡 partial mass 才花 GPU（且 ≥3 seeds / 更大 n 补功效）。
  - **(b) 条件化重述 + 发负/条件结果**（低成本无 GPU，安全兜底，与 (a) 配成 before/after）。
  - **(c) 转向直接测 composability/OOD-transfer**（真 pivot，成本高，(a) 也 null 时再上）。
